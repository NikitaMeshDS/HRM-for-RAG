# Hierarchical Reasoning Model (оригинал)

**Авторы:** Guan Wang, Jin Li, Yuhao Sun, Xing Chen, Changling Liu, Yue Wu, Meng Lu, Sen Song, Yasin Abbasi Yadkori
**Организация:** Sapient Intelligence (Singapore) + Tsinghua University
**Дата:** 26 июня 2025 (v3: 4 августа 2025)
**Ссылка:** https://arxiv.org/abs/2506.21734
**Код:** https://github.com/sapientinc/HRM

---

## TL;DR

Двухуровневая рекуррентная архитектура (27M параметров), обученная на ~1000 примеров без
pretraining и без CoT-данных, решает задачи (судоку, лабиринты, ARC-AGI), на которых
современные CoT-модели полностью проваливаются (0% точности). High-level модуль отвечает за
медленное абстрактное планирование, low-level — за быстрые детальные вычисления.

---

## Проблема

Трансформеры имеют фиксированную вычислительную глубину — это математически ограничивает их
(классы сложности AC⁰/TC⁰), не позволяя выполнять произвольно сложные алгоритмические
рассуждения end-to-end. Chain-of-Thought — костыль: хрупкая декомпозиция, требует много данных,
медленная генерация.

## Идея

Три принципа из нейронауки, перенесённые в архитектуру:
- **Hierarchical processing** — высокоуровневые зоны интегрируют информацию дольше, низкоуровневые
  обрабатывают быстро и детально
- **Temporal separation** — разные таймскейлы (мозговые тета/гамма-волны)
- **Recurrent connectivity** — обратные связи для итеративного уточнения

## Архитектура

4 компонента: input network, L-module (low-level), H-module (high-level), output network.

За один "high-level cycle" L-модуль делает T шагов, обновляясь на основе своего состояния и
**замороженного** состояния H. После T шагов H-модуль делает **один** шаг, забирая финальное
состояние L. Затем L "перезапускается" в новом контексте от H.

**Hierarchical convergence** — ключевой механизм: обычные RNN быстро сходятся к неподвижной точке
и "затухают". Здесь H периодически "перезапускает" L, не давая ему застыть — эффективная глубина
N×T шагов вместо T.

**Обучение без BPTT** — one-step gradient approximation через Implicit Function Theorem: градиент
считается только через последний шаг каждого модуля (не разворачивая всю историю). Память O(1)
вместо O(T).

**ACT (Adaptive Computational Time)** — Q-learning над эпизодическим MDP решает, когда "хватит
думать" ("halt") а когда продолжать. Модель может масштабировать inference-time compute без
дообучения — обученная с max 8 сегментами, при inference с 16 сегментами продолжает улучшаться.

## Результаты

| Задача | HRM (27M) | CoT-модели |
|--------|-----------|------------|
| Sudoku-Extreme Full | почти идеально | 0% (полный провал) |
| Maze-Hard (30×30) | почти идеально | 0% |
| ARC-AGI | 40.3% | o3-mini-high 34.5%, Claude 3.7 (8K) 21.2% |

Обучение — всего ~1000 примеров на задачу, без pretrain, без CoT-разметки.

## Интерпретируемость

Разные стратегии для разных задач: в лабиринте — параллельное исследование путей с отсечением
тупиков; в судоку — depth-first search с бэктрекингом; в ARC — инкрементальные правки
(hill-climbing). Модель сама выучила разные алгоритмы под разные типы задач.

## Связь с мозгом

Participation Ratio (эффективная размерность представлений) у **обученной** модели: H-модуль
~90, L-модуль ~30 — прямо как иерархия размерности в реальной коре мозга мыши. У **необученной**
модели такой иерархии нет — значит это выученное свойство, не архитектурный артефакт.

## Связь с проектом HRM for RAG

Ключевая асимметрия для проекта: HRM силён в reasoning, слаб в фактических знаниях (не
использует pretraining/CoT данные вообще). RAG вставляет знания снаружи, не трогая reasoning-
механизм. НО: оригинальный HRM тестировался только на закрытых структурированных задачах
(судоку/лабиринты/ARC) — открытый вопрос, перенесётся ли преимущество на multi-hop QA с
естественным языком и внешним retrieval.

**Важно:** для языковых задач нужен HRM-Text (см. 02-hrm-text.md) — оригинальный репозиторий не
предназначен для текстовых RAG-задач напрямую.
